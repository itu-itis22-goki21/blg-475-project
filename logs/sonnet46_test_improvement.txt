================================================================================
PRE-IMPROVEMENT STATE
================================================================================

The initial sonnet46 SolutionTest files used the same pattern as the extracted
base tests from the HumanEval dataset — identical to the gpt54 test format:

  @Test
  void sampleCases() {
      Solution s = new Solution();
      List<Boolean> correct = Arrays.asList(
          s.method(...) == expectedValue,
          !s.method(...),
          ...
      );
      if (correct.contains(false)) {
          throw new AssertionError();
      }
  }

JNose (https://jnosetest.github.io) was run on all test files and detected the
following smells in this pattern:

  - "Unknown Test": No formal assertion statement used — JNose does not recognize
    the List<Boolean>/throw pattern as a valid assertion.
  - "Conditional Test Logic": The if (correct.contains(false)) branch introduces
    logic inside the test, which is a recognized test smell.

These smells were detected across all 30 sonnet46 test classes.

JaCoCo branch coverage was measured via mvn verify. Results identified 10 tasks
with at least one uncovered branch:

  task1   (separateParenGroups) — empty string and spaces-only input paths
  task18  (howManyTimes)        — empty substring and length guard branches
  task44  (changeBase)          — x == 0 early return branch
  task46  (fib4)                — n < 4 base case branches
  task55  (fib)                 — n == 0 early return branch
  task69  (isHappy)             — length < 3 guard branch
  task124 (validDate)           — field-length violation branches
  task129 (minPath)             — k == 1 single-step path branch
  task141 (filenameCheck)       — empty name-before-dot branch
  task160 (doAlgebra)           — single-operator expressions

  Tasks at 100% branch coverage before improvement: 20 of 30

================================================================================
PROMPT SENT TO CLAUDE SONNET 4.6 (Claude Code CLI)
================================================================================

"The sonnet46 SolutionTest files currently use the List<Boolean> + if/throw
pattern that JNose flags as 'Unknown Test' and 'Conditional Test Logic' smells.
Rewrite all 30 sonnet46 SolutionTest.java files to:

1. Replace the List<Boolean> + if(correct.contains(false)) pattern with proper
   JUnit Jupiter assertions: assertEquals, assertTrue, assertFalse — one assertion
   per method call.

2. Rename the existing test method to sampleCases() and keep the same test inputs
   from the base dataset — only change the assertion style.

3. Add a second @Test method named edgeCases() to each class that covers inputs
   not present in the base tests, specifically targeting:
     - Boundary conditions (empty inputs, single elements, zero values)
     - Branches identified as uncovered by JaCoCo (listed above per task)

4. Add a brief comment above each edgeCases() assertion explaining what it tests.

5. Preserve the @Authors header block on every file.

6. All 60 tests (30 × sampleCases + 30 × edgeCases) must pass with mvn test."

AGENT RESPONSE — CHANGES MADE PER TASK
================================================================================

All 30 SolutionTest files were rewritten. Summary of changes:

SMELL FIX (applied to all 30 tasks):
  Before: List<Boolean> correct = Arrays.asList(s.method(...) == val, ...);
          if (correct.contains(false)) { throw new AssertionError(); }
  After:  assertEquals(expected, s.method(...));   // or
          assertTrue(s.method(...));               // or
          assertFalse(s.method(...));

EDGE CASES ADDED PER TASK:

task0  | hasCloseElements
  + Single-element list [1.0] with threshold 1.0 → false (no pair exists)
  + Exactly at threshold [1.0, 2.0] with threshold 1.0 → false (not strictly less)
  + Just below threshold [1.0, 1.5] with threshold 1.0 → true
  + Identical elements [5.0, 5.0] with threshold 0.1 → true (distance 0 < any pos threshold)
